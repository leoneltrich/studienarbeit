Furthermore, the \texttt{preloadModels} method guarantees that heavy machine learning assets are loaded into RAM during the boot or JIT phase, thereby preserving sub-second query latency during actual execution (NFR-1, NFR-2). Additionally methods are provided to manage optimized catalog instances, which allows an engine to store the catalog in it's preprocessed engine specific form. This reduces computational overhead during execution, contributing to the sub-second query latency. The \texttt{toolWeight} and \texttt{optionWeight} methods allow to give more accurate engines a higher importance during the ranking of candidates.
