\section*{Abstract}

Navigating complex command-line interface (CLI) syntax remains a challenge for developers, yet resolving this through cloud-based AI assistants introduces severe privacy risks in secure environments. This thesis addresses this trade-off by designing and implementing a local-first, CPU-efficient CLI command option retrieval system that operates entirely offline. Built in Rust using Hexagonal Architecture, the system combines a quantized bi-encoder embedding model (\texttt{embeddinggemma-300M-Q8\_0}) with a BM25 keyword search, backed by a Transition-Driven Command Assembly and Validation engine (STRS). To run efficiently on standard CPU hardware without GPU acceleration, the binary is compiled targeting musl libc with hardware-level SIMD optimizations. 

Technical evaluation ($N=609$ queries) demonstrates high resource efficiency, delivering an average execution latency of 563.21~ms and a stable memory footprint of 658.62~MB. In terms of retrieval, the model achieves 80.30\% accuracy in identifying the target command-line utility. When the correct utility is selected, the validation engine blocks 39.06\% of commands due to option conflicts; when an incorrect utility is selected, command generation is blocked in 53.33\% of cases due to option conflicts. While only 11.82\% of all queries result in a command generated for the correct utility with all required options matched, among commands generated for the correct utility, 24.16\% contain all required options, 51.01\% match a subset, and 24.83\% contain none. Globally, silently incorrect recommendations occur in 21.35\% of cases (9.20\% wrong tool generated, and 12.15\% correct tool with zero option matches). 

An exploratory usability study ($N=9$) indicates high user satisfaction, showing a clear learning effect as the median completion time fell from 603~seconds for the first task to 214~seconds for the third task, while demonstrating the system's potential to assist users with unfamiliar, proprietary tools. Ultimately, while retrieval accuracy is acceptable but not yet optimal, these findings demonstrate that local-first command retrieval is computationally feasible, offering secure development environments a viable, privacy-preserving alternative to cloud-based assistants.

\section*{Abstrakt}

Obwohl die Kommandozeile (CLI) ein mächtiges Werkzeug ist, stellt ihre komplexe Syntax Anwender oft vor Hürden. Herkömmliche Hilfestellungen wie cloudbasierte KI-Assistenten scheiden hierbei in sicherheitskritischen oder isolierten Umgebungen aus Datenschutzgründen aus. Diese Arbeit untersucht eine datenschutzkonforme Alternative und präsentiert ein lokal ausführbares, CPU-effizientes CLI-Hilfesystem, das vollständig offline arbeitet. Der in Rust als hexagonale Architektur implementierte Prototyp kombiniert ein quantisiertes semantisches Embedding-Modell ($\texttt{embeddinggemma-300M-Q8\_0}$) mit einer lexikalischen BM25-Suche. Eine integrierte Validierungskomponente (STRS) prüft Optionseingaben, um die Einhaltung von Syntaxregeln zu garantieren. Um eine flüssige Ausführung auf Standard-CPUs ohne Grafikbeschleunigung zu ermöglichen, ist die Binärdatei statisch gegen die musl-Bibliothek gelinkt und mit hardwarenahen SIMD-Optimierungen kompiliert. 

In einer technischen Evaluierung ($N = 609$ Anfragen) erweist sich das System als äußerst ressourcenschonend: Die durchschnittliche Latenz liegt bei 563,21~ms bei einem konstanten Arbeitsspeicherbedarf von rund 658,62~MB. Das System ordnet 80,30\% der Anfragen dem richtigen CLI-Tool zu. Bei korrekter Zuordnung fängt die Validierung in 39,06\% der Fälle widersprüchliche Flag-Kombinationen ab. Wird das falsche Tool zugeordnet verhindert die Validierung in 53,33\% der Fälle eine fehlerhafte Befehlsgenerierung. Insgesamt führten 11,82\% aller Testanfragen direkt zu einem Befehl, der das korrekte Tool sowie alle erforderlichen Flags enthielt. Betrachtet man allerdings nur die erfolgreich generierten Befehle für das korrekte Tool, so enthielten 24,16\% alle erforderlichen Optionen, 51,01\% eine Teilmenge und 24,83\% keine. Global treten unbemerkt fehlerhafte Empfehlungen in 21,35\% aller Fälle auf, aufgeteilt in 9,20\% für ein falsches Tool und 12,15\% für das korrekte Tool ohne passende Flags. 

Eine explorative Usability-Studie ($N=9$) bescheinigt dem System eine hohe Benutzerzufriedenheit. Dabei zeigte sich ein klarer Lerneffekt, da die durchschnittliche Bearbeitungszeit von der ersten zur dritten Aufgabe von 603 auf 214~Sekunden sank. Die Evaluierung verdeutlicht zudem das Potenzial des Systems, Anwender beim Umgang mit unbekannten, proprietären Tools zu unterstützen. Schließlich belegen diese Ergebnisse, dass es technisch machbar ist auf Standardhardware, CLI-Tools und deren Flags auf Basis natürlicher Sprache zu ermitteln.

wodurch sicheren Entwicklungsumgebungen eine tragfähige, datenschutzkonforme Alternative zu cloudbasierten Assistenten geboten wird, auch wenn die Abrufgenauigkeit akzeptabel, aber noch nicht optimal ist.
