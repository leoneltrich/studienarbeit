\section*{Abstract}

Resolving complex command-line interface (CLI) syntax via cloud-based assistants introduces severe privacy risks in secure development environments. This thesis presents a local-first, offline alternative: a CPU-efficient command option retrieval system. Built in Rust using Hexagonal Architecture, the stateless tool combines a quantized embedding model (\texttt{embeddinggemma-300M-Q8\_0}) with BM25 keyword search, backed by a syntax validation engine (STRS). It is compiled with hardware-level SIMD optimizations for efficient CPU-only execution. 

Technical evaluation (N = 609 queries) confirms high efficiency, averaging 563.21~ms latency and a 658.62~MB memory footprint. The system achieves 80.30\% accuracy in identifying the target command-line utility. When the correct utility is selected, the validation engine aborts 39.06\% of generations due to option conflicts; when an incorrect utility is selected, command generation is aborted in 53.33\% of cases due to option conflicts. 11.82\% of all queries result in a command generated with the correct tool and all required options. However, if the analysis is limited to successfully generated commands where the correct tool was picked, 24.16\% contain all required options, 51.01\% match a subset, and 24.83\% contain none. Globally, incorrect recommendations occur in 21.35\% of cases (9.20\% wrong tool generated, and 12.15\% correct tool with zero option matches). 

An exploratory usability study ($N=9$) indicates high user satisfaction. A clear learning effect is shown as the median completion time fell from 603~seconds for the first task to 214~seconds for the third, while demonstrating the system's potential to assist users with unfamiliar, proprietary tools. Ultimately, while retrieval accuracy is acceptable but not yet optimal, these findings demonstrate that local-first command retrieval is computationally feasible, offering secure environments a viable, privacy-preserving alternative to cloud-based assistants.

\section*{Abstrakt}

Das Nutzen von Cloud-basierten KI-Assistenten stellt Kommandozeilenanwender vor Datenschutzprobleme. Diese Arbeit untersucht eine datenschutzkonforme Alternative und präsentiert ein lokales, CPU-effizientes Tool, das offline arbeitet. Der in Rust als hexagonale Architektur implementierte Prototyp kombiniert ein quantisiertes Embedding-Modell ($\texttt{embeddinggemma-300M-Q8\_0}$) mit einer lexikalischen BM25-Suche. Eine integrierte Validierungskomponente (STRS) prüft Optionseingaben, um die Einhaltung von Syntaxregeln zu garantieren. Eine flüssige Ausführung auf Standard-CPUs wird durch statisches Linking und eine Kompilierung mit hardwarenahen SIMD-Optimierungen ermöglicht.

Die technische Evaluierung ($N = 609$ Anfragen) zeigt, das System ist äußerst ressourcenschonend: Die durchschnittliche Latenz liegt bei 563,21~ms bei einem konstanten Arbeitsspeicherbedarf von 658,62~MB. Das System ordnet 80,30\% der Anfragen dem richtigen CLI-Tool zu. Bei korrekter Zuordnung fängt die Validierung in 39,06\% der Fälle widersprüchliche Flag-Kombinationen ab. Wird das falsche Tool zugeordnet verhindert die Validierung in 53,33\% der Fälle eine fehlerhafte Befehlsgenerierung. Insgesamt führten 11,82\% aller Testanfragen direkt zu einem Befehl, der das korrekte Tool sowie alle erforderlichen Flags enthielt. Betrachtet man nur die erfolgreich generierten Befehle mit korrekt gewähltem Tool ergibt sich, dass 24,16\% alle erforderlichen Optionen, 51,01\% eine Teilmenge und 24,83\% keine davon enthielten. Global treten unbemerkt fehlerhafte Empfehlungen in 21,35\% aller Fälle auf (9,20\% falsches Tool, 12,15\% korrektes Tool ohne passende Flags).

Eine explorative Usability-Studie ($N=9$) bescheinigt dem System eine hohe Benutzerzufriedenheit. Durch die Gewöhnung an das Tool sank die mediane Bearbeitungszeit von der ersten zur dritten Aufgabe deutlich von 603 auf 214~Sekunden. Zudem zeigt die Evaluierung, dass das System Anwender beim Umgang mit unbekannten, proprietären Werkzeugen effektiv unterstützt. Insgesamt verdeutlicht der Prototyp, dass ein lokaler und datenschutzkonformer Abruf von CLI-Optionen auf Standardhardware technisch wie praktisch realisierbar ist. Auch wenn die Abrufgenauigkeit noch Raum für Verbesserungen bietet, stellt das System eine vielversprechende Offline-Alternative zu cloudbasierten Assistenten in sicherheitskritischen Umgebungen dar.
