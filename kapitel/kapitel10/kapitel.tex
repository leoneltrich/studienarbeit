\section{Conclusion And Future Work}

The thesis successfully answers the research questions. A functional PoC system was designed that operates within the required constraints. Whilst only retrieving the accurate options in a fraction of absolute cases it retrieved a subset of the required options much more frequently and successfully failed command generation if conflicting options were selected. This lead to only about 21.4\% of cases where no correct options were suggested at all with only 9.2\% confidently suggesting a wrong tool without causing errors. This shows the potential of the system especially as a tool for learning if suggested options always were suggested with a fitting explaination to allow the user to verify the suggested command actually does what they want. So while this tool will not help at solving the problem of poor documentation it can aid in assisting the understanding and simple utilization of comand line tools without requiring users to read man pages.

Future works should evaluate the performance of Specialized neural networks as a substitutional engine and maybe also the performance of cross encoders as a reranker for the ranked options, using a cross encoder as a direct retrieval engine might lead to longer response times and is not recommended only alternative would be to build the cross encoder directly into the other retrieval engines but this might lead to increased load times and should only done with careful consideration. Furthermore experiments to build a Classifier model that detects false matches might significantly increase performance. It is recommended to keep improving this approach by adding and determining the missing weights to the individual engines. Finally a more broad user study with a representative statistically reliable cohort should be conducted. 
Furthermore soem of the results of this work are directly transferable to the implementation and research of creating localized RAG pipelines with strict performance boundries since essentially the proposed system implements RAG just without utilizing a LLM for the final command generation.
