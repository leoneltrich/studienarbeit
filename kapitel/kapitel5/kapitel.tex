\section{Systems Design And Implementation}

\subsection{Design}

Database:
- Tabelle für Tool beschreibungen
    - Tool name (ID)
    - Tool Beschreibung (TEXT)
    - Tool Beschreibung Vector (BLOB)
- Tabelle für tool option Beschreibungen
    - option (ID)
    - Tool Name (ID)
    - Tool Option beschreibung (TEXT)
    - Tool Option Beschreibung Vector (BLOB)

Pipeline Design

\subsubsection{Requirements}

- Niedrige Latenz
- Ram <1GB
- 85 Prozent akkurat
- Schneller Startup
- Keine Internetanbindung nötig (Läuft lokal)
- (Einfach) Erweiterbar
- Embeddings der Tools müssen gespeichert werden dass sie nicht die ganze zeit berechnet werden müssen
- Die rohen beschreibungen müssen gespeichert werden
- Daten müssen gespeichert werden ohne anbindung an ressourcen intensiven Datenbank server
- Es muss eine Validierung der Command syntax geben
- Das Embedding modell muss austauschbar sein um in zukunft ein anderes verwenden zu können
- Es soll eine kompiliere Sprache genutzt werden um schnelle Startup zeiten und portabilität zu gewährleisten
